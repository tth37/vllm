════════════════════════ Analysis ════════════════════════

1. TP=1 Baseline (Host vs Docker vs DockerBE):
   - All three TP=1 configs achieve nearly identical throughput (~830 tok/s output)
   - Docker container overhead is negligible for TP=1 (3% TPOT increase)
   - Docker backend adds ~28% TPOT increase due to TCP MessageQueue overhead

2. TP=2 Comparison:
   - Host MP: Median TPOT 5.39ms (baseline, async scheduling enabled)
   - Docker Container MP: Median TPOT 6.16ms (+14%, container runtime overhead)
   - Docker Backend: Median TPOT 6.65ms (+23%, TCP MQ + sync scheduling)

3. Overhead Breakdown (see profile_report.txt for deep-dive):
   - NCCL_DEBUG=INFO: 0% impact (confirmed by A/B test)
   - Docker container runtime: ~0.77ms (+14%)
   - TCP MessageQueue + sync scheduling: ~0.49ms (+9%)
   - Total DockerBE overhead vs Host MP: ~1.26ms (+23%)

4. Key Finding:
   - The initial Exp 6 showing 13-17ms TPOT was due to external GPU contention
   - Under clean conditions, DockerBE overhead is modest (23%)
   - For larger models where inference dominates (>100ms per step),
     the 1.3ms control plane overhead would be <1.3% — negligible
